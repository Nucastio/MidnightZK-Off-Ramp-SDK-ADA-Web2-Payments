# Changelog

All notable changes to the **MidnightZK Off-Ramp SDK (ADA ⇆ Web2 Payments)** are documented here. Versioning follows [SemVer](https://semver.org).

## [1.0.0] — Public release

First publicly tagged release of the MidnightZK Off-Ramp SDK. Code surface is stable; the entire integration path is exercised by automated tests and real Cardano Preprod + Wise sandbox runs.

### Released artefacts

- **Tagged release:** [v1.0.0](https://github.com/Nucastio/MidnightZK-Off-Ramp-SDK-ADA-Web2-Payments/releases/tag/v1.0.0) — source archive attached.
- **Hosted developer docs:** <https://nucastio.github.io/MidnightZK-Off-Ramp-SDK-ADA-Web2-Payments/> — built by `.github/workflows/docs.yml` on every push to `docs/`.
- **MIT LICENSE** at the repo root.

### Added

- **`OffRampSDK` class** (`sdk/src/sdk.ts`) wiring the four off-chain steps (`initiateOffRamp` → `generateZKProof` / `verifyZKProof` → `submitPayment` → `confirmSettlement`) with typed inputs/outputs.
- **Cardano helpers** (`sdk/src/cardano/`): `createAppLucid`, `submitLockTx`, `submitReleaseTx`, `submitRefundTx`, `escrowScript`, `escrowScriptAddress`, `escrowDatumCbor`, `RELEASE_REDEEMER`, `REFUND_REDEEMER` — built on Lucid Evolution.
- **Cardano PlutusV3 escrow validator** (`cardano/escrow/validators/escrow.ak`, Aiken v1.1.21) — Preprod script `addr_test1wrvzmkxhfmr9j0u8g6p4cpkevqja4tn8qr88z7l7nc2tqrsxln25g`; 5 real Preprod tx hashes recorded in [`docs/testnet-evidence.md`](docs/testnet-evidence.md).
- **Midnight Compact circuit** (`contract/src/offramp.compact`) with predicates `provePayeeBinding`, `proveAmountBinding`, `proveComplianceFlag`, `proveOffRampSettlement`; 4 SNARK proofs deployed across blocks 15774→15784.
- **Rail adapters** (`sdk/src/adapters/`): Cash App (Afterpay sandbox semantics), Wise (live sandbox), Revolut. Behind a stable `RailAdapter` interface; mode toggled by `RAIL_ADAPTER_MODE=mock|sandbox`.
- **Settlement Oracle** (`sdk/src/oracle/settlement-oracle.ts`) — Ed25519-signed canonical attestations bound to `intent_id`; adapter webhook HMAC verification.
- **Hono backend** (`backend/api/`) + OpenAPI 3.0.3 spec + Swagger UI at `/docs`.
- **Off-ramp UI** (`ui/`) — vanilla HTML/CSS/JS with the 6-step pipeline view.
- **Internal test harness** (`scripts/internal-test.ts`, `scripts/internal-test-lib.ts`) — 30 simulated off-ramps × 3 rails, regenerates [`docs/internal-testing-report.md`](docs/internal-testing-report.md).
- **Preprod runner scripts** (`scripts/preprod-{lock,refund,release}.ts`) — drive real Cardano Preprod LOCK / REFUND / RELEASE flows.
- **MkDocs Material documentation site** with integration guide, API reference, SDK reference, examples, architecture, and the consolidated [Final Testing & Release](docs/final-testing-and-release.md) document.
- **GitHub Actions** — `.github/workflows/docs.yml` (Pages deploy) + `.github/workflows/release.yml` (tag-triggered GitHub Release).
- **Specifications committed at repo root:** Project Initiation, SRS, TAD v1.1 (canonical), TAD v1 (superseded).

### Changed

- **`docs/internal-testing-report.md`** regenerated by the v1 pre-release harness run (`2026-06-13`): 30 simulated off-ramps, 96.7% success (≥ 90% threshold), avg prove **744.6 ms** (≤ 50 000 ms NFR-2 budget). The single failure is the intentional negative-path that exercises the adapter-rejection branch.

### Security

- **No secrets in the tree.** `.env` is gitignored and not tracked; `.env.example` documents every required variable. `OPERATOR_ED25519_SK_HEX`, `BLOCKFROST_PROJECT_ID`, and all wallet mnemonics fail-closed with no defaults.
- **Settlement Oracle key** is operator-owned (`OPERATOR_ED25519_SK_HEX`); consumers verify attestations using only the public key returned by `operatorPublicKeyHex()`.

### Known notes

- **TryCloudflare URLs are eval-window only** — the live MVP URLs referenced in the README are ephemeral quick-tunnels tied to a local process; to reproduce, follow the [quickstart](docs/quickstart.md).
- **Wise sandbox `/payments`** call is SCA-gated by the provider — see [`docs/sandbox-evidence/README.md`](docs/sandbox-evidence/README.md) §"Step 5 note". The SDK's responsibility ends at submitting the funding intent.
